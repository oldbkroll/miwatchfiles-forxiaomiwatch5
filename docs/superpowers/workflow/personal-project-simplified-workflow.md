# WatchFiles 个人小项目简化流程

这套流程用于单人、小步迭代、真机优先的阶段开发。目标是保留文件安全和可追溯性，同时避免把每个小任务做成大型项目流程。

## 阶段与会话

- 一个阶段使用一个独立会话；阶段完成后关闭会话并从新的阶段会话继续。
- 新会话只读取入口、长期上下文、总路线图、当前测试清单，以及当前阶段的 spec/plan/checkpoint。
- 已关闭阶段的逐任务 brief、review diff 和临时日志不作为默认上下文。

## 实现与审查

- 文件安全核心逻辑保留 TDD：先写一个能复现风险的失败测试，再做最小修复，再跑全量 Debug 单测。
- 复制、移动等高风险增量在检查点做一次轻量复审；取消逐任务双审、超详细计划和过密提交。
- 不做与当前阶段无关的重构；每个提交应对应一个可解释的增量或阶段收尾。

## 构建与真机

- 开发期间只构建和安装 Debug；整个阶段完成时才统一构建 Release/阶段包。
- 真机写操作只允许在 `/storage/emulated/0/Download/WatchFilesTest/M1Sandbox`。
- 无线 ADB 每次动态发现 `M2505W1/grasslte` 的在线序列号，不假定历史地址；设备多次掉线时暂停并请用户重新打开无线调试。
- 真机验收必须记录界面结果、源/目标哈希、源删除状态、任务临时残留和 `AndroidRuntime` 日志。

## 阶段收尾

阶段完成前统一执行：

1. Debug 单测、`assembleDebug`、`lintDebug` 和 `git diff --check`。
2. 更新当前 checkpoint 和路线图状态。
3. 提交代码、阶段文档和必要的测试记录。
4. 在新会话开头只从新的阶段 checkpoint 接续，不重复回放整个历史会话。

安全底线保持不变：生产移动/发布路径不得使用 `ATOMIC_MOVE` 或 `REPLACE_EXISTING`，不得先删除未经确认的旧目标。
