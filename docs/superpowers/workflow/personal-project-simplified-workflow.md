# WatchFiles 个人小项目简化流程

这套流程用于单人、小步迭代、真机优先的阶段开发。流程强度按风险分级：外围体验和只读能力快速迭代，文件安全和真实写入保留完整证据。

## 阶段与会话

- 一个阶段使用一个独立会话；阶段完成后关闭会话并从新的阶段会话继续。
- 新会话读取 `README.md`、项目上下文、路线图、当前测试清单和当前阶段 spec/plan/checkpoint；再按当前任务风险读取相关 workflow 文档。
- 不重新回放已关闭阶段的逐任务 brief、review diff 和临时日志。
- 每个提交对应一个完整功能、可解释修复或阶段收尾，不要求每个微任务单独提交。

## 风险分级

### A 级：低风险改动

适用于文档、文案、颜色、间距和不改变行为的重命名。

- 用几行说明目标和验收条件。
- 不强制先写测试；按需做 Markdown、静态检查或手工检查。
- 文档-only 改动不构建 APK。
- 不要求独立复审和真机写操作。

### B 级：中风险改动

适用于启动优化、文本查看、页面导航、列表交互、表冠交互、MIME 判断、图片查看和不涉及真实写入的媒体能力。

- 用 3-5 行小计划说明目标、范围和验收条件，不强制拆成完整 spec、plan 和逐任务 brief。
- 关键状态转换和容易回归的逻辑先补单测；普通 UI 按需要补测试。
- 开发中运行聚焦测试、静态检查或手工验证。
- 涉及手表交互时做一次动态 ADB 真机只读检查，不创建、删除或修改真实文件。
- 跨模块或阶段性功能才安排独立复审；阶段收尾统一运行完整 Debug gate。

文本查看属于 B 级，但文本覆盖和当前目录内另存为会修改真实文件，属于 C 级。

### C 级：高风险改动

适用于文本覆盖/另存为、DELETE、COPY、MOVE、权限、路径安全、取消、失败恢复、临时文件、前台文件操作服务、真实设备写入和 ZIP 解压。

- 保留设计确认、TDD、故障注入测试和必要的独立复审。
- 真机写操作严格限制在 `/storage/emulated/0/Download/WatchFilesTest/M1Sandbox`。
- 每次设备会话动态发现无线 ADB 在线 serial，不复用历史地址。
- 记录界面结果、源/目标哈希、源删除状态、任务临时残留和 crash audit。
- 阶段结束前运行完整 Debug 单测、`assembleDebug`、`lintDebug` 和 `git diff --check`。
- ZIP 解压必须额外验证路径穿越、同名覆盖、空间不足、取消、部分成功和临时目录清理。

## 构建与真机

- 开发期间只构建和安装 Debug；阶段完成后才进行 Release 交接。
- 无线 ADB 每次动态发现 `M2505W1/grasslte` 的在线序列号，不假定历史地址；设备多次掉线时暂停并请用户重新打开无线调试。
- B 级只读真机检查和 C 级写操作验收都必须记录实际结果，不把未执行的设备行为写成已验收证据。
- 真机写操作不触碰 system、media、个人目录或 M1Sandbox 之外的路径。

## 阶段收尾

阶段完成前统一执行：

1. Debug 单测、`assembleDebug`、`lintDebug` 和 `git diff --check`。
2. 更新当前 checkpoint、测试清单和路线图状态。
3. 提交代码、阶段文档和必要的测试记录。
4. 在新会话开头从新的阶段 checkpoint 接续，不重复回放整个历史会话。

安全底线保持不变：生产移动/发布路径不得使用 `ATOMIC_MOVE` 或 `REPLACE_EXISTING`，不得先删除未经确认的旧目标；不改变 `targetSdk 29`、`requestLegacyExternalStorage`、`armeabi-v7a` 或小米手表自定义表冠滚动兼容逻辑。

`personal-project-simplified-workflow.md` 是正式执行规则；`2026-07-17-workflow-simplification-memo.md` 是建议备忘录，不替代本文件，也不改写已完成阶段历史。
