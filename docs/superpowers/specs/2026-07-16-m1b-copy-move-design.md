# WatchFiles M1B 安全复制与移动设计

日期：2026-07-16
状态：设计已由用户逐节批准

## 1. 目标与交付方式

M1B 为 WatchFiles 增加前台单任务的文件与目录复制、移动能力，首要目标是在取消、失败和同名替换过程中保护源文件及既有目标。

M1B 分两个可独立真机验收的检查点交付：

1. 安全复制：目标目录选择、递归复制、扫描与进度、取消、`.part` 清理、非法目标拦截，以及同名项目的“替换全部”或“取消任务”。
2. 安全移动：同一文件系统的安全快速移动，以及无法安全快速移动时的“完整复制并确认目标成功后才删除源文件”回退路径。

每个检查点只运行单元测试、Debug 构建、Lint 和真机 Debug 验收。M1B 不构建 Release，不更新阶段发布包或正式版本号。

本文是 `2026-07-15-m1-file-operations-design.md` 的 M1B 细化规范。若两份文档在 M1B 冲突选项上不一致，以本文为准：只保留“替换全部”和“取消任务”。

## 2. 范围边界

- 只保证应用进程存活期间的前台任务运行。
- 不使用 WorkManager 或前台服务，不实现息屏续传、进程恢复或任务恢复日志；这些能力属于 M3。
- 同一时间最多运行一个复制或移动任务。
- M1B 不实现删除操作；永久删除和二次确认属于 M1C。
- 同名冲突只提供“替换全部”和“取消任务”，不提供保留两者、跳过或目录合并。
- 取消时保留已经完整发布成功的项目，只清理尚未发布且能由当前任务 ID 确认拥有的临时项目。

## 3. 架构

### 3.1 `FileOperationEngine`

新增独立操作引擎，负责：

- 预扫描源文件与目录树
- 统计可读取的项目数和总字节数
- 校验源路径、目标路径和路径包含关系
- 递归复制文件与目录且不跟随符号链接
- 创建、同步、发布并清理带任务 ID 的 `.part` 临时项目
- 执行同存储快速移动和复制后删除的回退移动
- 在固定大小的数据块之间检查协作式取消
- 将文件系统异常转换为结构化结果

引擎不直接依赖 Compose 或页面导航，也不维护浏览列表状态。

### 3.2 `FileOperationCoordinator`

新增协调器，负责：

- 强制单任务执行
- 为任务生成唯一 ID
- 暴露扫描、运行、等待冲突确认、取消中和最终结果状态
- 首次冲突时暂停任务，等待“替换全部”或“取消任务”
- 记录用户选择；选择替换后，本任务后续所有冲突直接采用替换策略
- 向操作引擎传递取消请求

协调器使用协程和 `StateFlow` 暴露状态，不把长任务塞入现有 `FileBrowserViewModel`。

### 3.3 浏览与目标目录选择

`FileBrowserViewModel` 继续负责目录、隐藏文件和选择状态。它只把选中源路径、操作类型和最终目标目录提交给协调器，并在任务结束后刷新目录、清除失效选择。

复制或移动入口进入独立的目标目录选择模式：

- 始终显示待处理项目数量
- 文件夹可以进入
- 普通文件不可打开或选择
- 提供“放到此处”和“取消”
- 保持现有自定义表冠滚动
- 所有关键按钮放在圆屏安全区域内，不使用小型弹窗

## 4. 操作状态与数据流

协调器至少公开以下状态：

- `Idle`：无任务
- `Scanning`：扫描项目、字节和目标合法性
- `Running`：包含操作类型、当前项目、已处理/总项目数、已处理/总字节数
- `WaitingForReplacement`：首个同名冲突暂停，等待替换全部或取消
- `Cancelling`：不再开始新项目，等待当前数据块结束并清理临时项目
- `Succeeded`：所有源项目处理完成
- `PartiallySucceeded`：已有目标成功，但旧目标备份或移动源删除失败
- `Failed`：任务失败，包含用户摘要和技术原因
- `Cancelled`：取消清理结束，包含取消前已经成功发布的项目数

数据流为：

1. 用户多选源项目并选择复制或移动。
2. 用户在目标目录选择模式中浏览并点击“放到此处”。
3. 协调器开始单任务，操作引擎预扫描并校验。
4. 引擎逐项执行，协调器向 UI 发布进度。
5. 首次同名时暂停；替换确认应用于本任务全部后续冲突，取消则进入协作式清理。
6. 任务结束后显示结构化结果。
7. 用户返回目录，浏览 ViewModel 刷新当前目录并退出选择状态。

运行期间再次启动复制或移动会被明确拒绝，不排入第二个队列。

## 5. 启动前校验

启动前拒绝以下情况：

- 源项目已经不存在或不可读
- 目标目录已经不存在或不可写
- 目标目录与源文件夹完全相同
- 目标目录位于任一待复制或移动文件夹内部
- 目标路径会产生明确路径循环
- 已有文件任务运行
- 可用空间明显小于可统计的待写入字节数

空间检查只作为预警。即使预检查通过，运行时仍必须正确处理空间不足和写入失败。

## 6. 安全复制

### 6.1 文件

目标文件先写到目标父目录中的隐藏临时文件：

`.目标名称.watchfiles-任务ID.part`

写入完成后刷新并同步文件内容，再发布为正式名称。写入、同步或发布失败时，源文件保持不变，并清理由当前任务 ID 创建的临时文件。

### 6.2 目录

目标目录先完整构建在目标父目录中的隐藏临时目录：

`.目标名称.watchfiles-任务ID.part-dir`

临时目录内递归复制完整目录树。所有子项目成功后才发布正式目录。取消或失败时，只递归清理由当前任务 ID 创建的临时目录；绝不清理既有目录或无法确认归属的项目。

### 6.3 同名替换

首次遇到同名文件、同名目录或文件/目录类型冲突时，任务进入 `WaitingForReplacement`。圆屏页面只提供：

- “替换全部”：确认后替换当前冲突，并自动替换本任务后续所有冲突
- “取消任务”：停止后续操作并清理尚未发布的临时项目

替换不允许先删除旧目标。引擎必须：

1. 在同一父目录完整生成带任务 ID 的新临时目标。
2. 把旧目标改名为带任务 ID 的备份。
3. 发布新目标。
4. 发布失败时恢复旧目标。
5. 发布成功后删除旧备份。

若旧备份删除失败，新目标保持发布，任务报告 `PartiallySucceeded` 并保留可识别备份，不把结果伪装成完整成功。

目录冲突不合并内容；替换意味着整个既有目标目录被新目录替代。

## 7. 安全移动

移动首先判断源和目标是否位于同一文件系统，并尝试平台提供的安全移动能力。快速移动必须遵守相同的同名替换和结果报告规则。

快速移动不可用时，按以下顺序执行：

1. 使用安全复制规则生成并发布完整目标。
2. 校验目标类型和可获得的基本尺寸信息。
3. 删除源项目。

只有第 1、2 步成功后才允许删除源项目。源删除失败时保留已经完成的目标副本，报告 `PartiallySucceeded`，不得删除目标以掩盖部分成功。

移动取消发生在目标发布前时保留源项目并清理临时目标；目标已经完整发布后发生取消时，不回滚已完成项目，并在结果中明确完成数量。

## 8. 进度与取消

操作页面显示：

- 操作类型
- 当前项目名称
- 已处理项目数和总项目数
- 已处理字节数和可统计总字节数
- 取消按钮

扫描阶段也允许取消。部分项目大小无法读取时，项目进度仍必须准确可用，字节总量显示为未知。

取消是协作式的：当前固定大小数据块结束后停止，不再开始下一个项目。协调器进入 `Cancelling`，引擎清理当前任务尚未发布的临时项目。已经完整发布成功的项目不回滚、不删除；结果页显示“已完成 N 项，任务已取消”。

## 9. 错误和结果

界面使用简洁中文区分：

- 空间不足
- 没有读写权限
- 源项目已消失
- 读取失败
- 写入失败
- 同步失败
- 正式发布失败
- 旧目标恢复失败
- 移动源删除失败
- 非法目标目录
- 已有任务运行

技术异常信息写入日志并保存在结构化失败结果中，不直接把长堆栈显示在圆屏页面。

最终结果显示成功、取消、失败或部分成功，以及成功和失败项目数。任何旧备份未清理或源删除失败都必须显示为部分成功。

## 10. 真机与自动化验收

所有写操作只在以下目录执行：

`/storage/emulated/0/Download/WatchFilesTest/M1Sandbox`

### 10.1 第一检查点：安全复制

自动化与真机覆盖：

- 空文件、普通文件和大文件
- 空目录、三层嵌套目录和混合目录
- 多选文件与文件夹
- 正常目标复制
- 目标位于源目录内部时启动前拒绝
- 同名文件、同名目录及文件/目录类型冲突
- 替换全部和取消任务
- 复制中取消后源文件不变
- 取消或失败后不存在属于本任务的未发布 `.part`
- 用户自己创建的普通 `.part` 文件不被清理
- 连续执行任务后内存没有持续上涨

### 10.2 第二检查点：安全移动

自动化与真机覆盖：

- 同存储安全快速移动
- 快速移动不可用时复制后删除源文件
- 目标发布失败时源文件保持不变
- 源删除失败时保留目标副本并报告部分成功
- 移动复制阶段取消不损坏源文件
- 移动完成后浏览目录和选择状态正确刷新

### 10.3 设备连接

小米 Watch 5 的无线调试开关和端口不稳定。每次真机操作前重新运行 `adb devices -l` 和 mDNS 发现，不固定 IP、端口或连接别名。若连接连续多次丢失或无法重新发现，立即请用户在手表上重新开启无线调试，不对其他网络 ADB 设备执行命令。

## 11. 保持不变的项目约束

- 保留 `targetSdk 29` 的小米手表旧式文件权限兼容逻辑。
- 保留 `requestLegacyExternalStorage` 和现有权限流程。
- 保留 `rotaryScrollableBehavior = null` 与自定义表冠滚动，避免厂商系统缺少 Google 触觉类导致崩溃。
- 文件操作和扫描不得阻塞主线程。
- APK 只打包 `armeabi-v7a`。
- 圆屏关键操作必须可通过触摸完成。
