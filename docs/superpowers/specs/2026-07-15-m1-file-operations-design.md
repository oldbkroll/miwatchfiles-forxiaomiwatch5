# WatchFiles M1 基础文件操作设计

日期：2026-07-15  
状态：设计已由用户逐节批准，等待书面规范复核

## 1. 目标

M1 为 WatchFiles 增加可在小米 Watch 5 圆屏上安全使用的基础文件操作：

- 长按多选与全选
- 新建文件夹
- 单项目重命名
- 文件和文件夹的递归复制、移动
- 文件和文件夹的递归永久删除
- 同名冲突处理
- 单任务进度、取消和错误报告
- `.part` 临时文件与失败清理
- 空间不足、权限失败和非法目标提示

M1 的首要原则是保护源文件。中断复制不得损坏源文件；移动只有在目标确认成功后才允许删除源项目。

## 2. 范围边界

M1 只保证应用进程存活期间的前台任务执行。息屏继续、前台服务、进程被杀后的任务恢复和恢复日志属于 M3，不在本设计中实现。

M1 不实现回收站。删除为永久删除，但必须经过独立确认页。内部存储根目录不可选择或删除。

M1 不并行执行文件任务。同一时间最多存在一个复制、移动或删除任务；用户必须等待、取消或处理完当前任务后才能开始下一个任务。

## 3. 架构

### 3.1 读取与写入职责分离

现有 `DirectPathRepository` 保持只读，只负责列出目录和生成 `FileEntry`。

新增 `FileMutationRepository`，负责：

- 校验名称和目标路径
- 新建文件夹
- 重命名单个文件或文件夹
- 提供操作引擎需要的目录建立、临时文件、改名和删除原语

新增 `FileOperationEngine`，负责：

- 预扫描递归项目数、可统计字节数和冲突
- 递归复制、移动和删除
- `.part` 写入、同步、正式改名和失败清理
- 进度上报与协作式取消
- 不跟随符号链接的边界保证

新增 `FileOperationCoordinator`，负责：

- 强制单任务执行
- 保存当前任务和总体进度状态
- 在遇到同名冲突时暂停并等待用户决定
- 将成功、部分成功、失败和取消结果暴露给 UI

浏览 ViewModel 继续管理当前目录，同时增加选择模式和选中路径集合。操作完成后由 ViewModel 刷新当前目录并清理失效选择。

### 3.2 操作状态

任务至少包含以下状态：

- `Idle`：没有任务
- `Scanning`：统计项目、字节和预检查
- `Running`：正在执行，包含当前项目、项目进度和字节进度
- `WaitingForConflict`：因同名目标暂停
- `Cancelling`：已请求取消，正在停止并清理本任务临时文件
- `Succeeded`：全部成功
- `PartiallySucceeded`：部分完成，例如跨目录移动已复制但源删除失败
- `Failed`：任务失败，包含失败项目和原因
- `Cancelled`：已取消并完成必要清理

所有状态必须能由 UI 明确显示，不允许把部分成功报告成完整成功。

## 4. 用户交互

### 4.1 多选

- 长按文件或文件夹进入选择模式并选中该项目。
- 选择模式下，单击项目切换选中状态，不再打开项目。
- 顶部显示“已选 N 项”。
- 操作入口包括全选、复制、移动、删除和取消。
- 仅选中一个项目时提供重命名。
- 内部存储根目录本身不作为可选项目。

### 4.2 新建与重命名

- 普通目录页提供“新建文件夹”。
- 通过系统输入控件调用设备已安装的搜狗手表输入法。
- 新建和重命名拒绝空名称、`.`、`..`、`/`、空字符以及当前文件系统无法接受的名称。
- 名称首尾空格不自动去除；提交前明确提示并要求用户修正，避免显示名称与实际名称不一致。
- 目标名称已存在时不直接覆盖，进入同名提示。

### 4.3 复制和移动目标选择

流程为：

1. 选择一个或多个文件、文件夹。
2. 点击复制或移动。
3. 进入目标文件夹选择模式。
4. 浏览到目标目录。
5. 点击“放到此处”或取消。

目标选择模式始终显示待处理项目数量。文件项目不可打开；文件夹仍可进入。以下目标必须在启动前拒绝：

- 与源项目完全相同的路径
- 位于待复制或移动文件夹内部的目录
- 不可写目录
- 会导致明确路径循环的目标

### 4.4 进度与结果

操作页显示：

- 操作类型
- 当前项目名称
- 已处理项目数与总项目数
- 已处理字节数与可统计总字节数
- 取消按钮

若部分项目大小无法读取，项目进度仍必须可用，字节总量显示为近似或未知。结果页列出成功、跳过和失败数量，并允许查看失败项目。

### 4.5 圆屏适配

选择操作、目标确认、冲突处理、删除确认和错误详情使用独立圆屏页面或安全区域内的列表，不使用容易被圆形边缘裁切的小型弹窗。所有关键操作均可通过触摸完成；目录浏览继续支持现有自定义表冠滚动。

## 5. 同名冲突

遇到同名目标时任务暂停，用户可选择：

- 保留两者：生成不冲突的新名称，例如 `名称 (1).扩展名`
- 替换：删除或替换目标前再次确认
- 跳过：保留现有目标并继续任务
- 取消任务：停止后续操作并清理本任务的临时文件

冲突页提供“应用到后续全部冲突”，但“替换全部”仍需一次明确的替换全部确认。

目录与文件类型冲突按同名冲突处理，不自动合并。首个 M1 版本不做目录内容合并；用户可选择保留两者、替换、跳过或取消。选择替换目录时，必须明确提示目标目录及其内容会永久删除。替换也必须先完整生成新目标，再发布新目标并清理旧目标，不能先删除旧目标再开始复制。

## 6. 文件安全规则

### 6.1 复制

文件复制到目标目录中的隐藏临时文件 `.目标名称.watchfiles-任务标识.part`。写入完成并同步成功后，再把临时文件改为正式名称。正式名称发布失败时，保留原目标不变并清理本任务临时文件。

目录复制先在目标父目录建立 `.目标名称.watchfiles-任务标识.part-dir` 临时目录，并在其中完整构建目录树，完成后再发布为正式目录。取消或失败时，只递归清理由本任务标识确认拥有的临时目录；绝不递归清理既有目录或无法确认归属的目录。

替换现有文件或目录时，先在同一父目录完整生成带任务标识的临时目标。发布时优先使用文件系统支持的原子替换；不支持时，把旧目标改名为带任务标识的备份，再发布新目标。发布失败必须恢复旧目标；发布成功后才删除备份。若备份删除失败，任务报告部分成功并保留可识别的备份，供用户确认后清理。

### 6.2 移动

同一文件系统且平台确认可安全执行时，允许使用原子移动快速路径。若快速路径不可用，则执行：

1. 按复制规则生成完整目标。
2. 校验目标类型和可用的基本尺寸结果。
3. 删除源项目。

源删除失败时结果为部分成功，并保留已完成的目标副本；不得删除目标以假装任务未发生。

### 6.3 删除

删除前扫描并显示项目数和可统计总大小。确认页明确写明“永久删除，无法恢复”。根存储目录不可删除。

删除文件夹采用深度优先：先删除子项目，再删除空目录。不跟随符号链接；链接本身按单个项目处理。

删除任务允许取消，但已删除项目无法恢复。确认页和操作页必须明确说明这一点。

### 6.4 取消与清理

取消是协作式的：当前小块写入结束后停止，不启动下一个项目。引擎删除本任务创建且尚未发布的 `.part` 文件，并按规则清理本任务新建的空目录。

应用下次启动或进入目录时，可以识别 WatchFiles 自己命名且带任务标识的遗留 `.part`。只有能够确认属于 WatchFiles 未完成任务的临时文件才允许提示清理，不能按扩展名批量删除用户自己的 `.part` 文件。

## 7. 错误处理

启动前尽量检查：

- 目标目录可写性
- 可用空间是否明显不足
- 源项目是否仍存在且可读
- 目标是否位于源文件夹内部
- 是否已有任务运行

运行中单独记录权限错误、空间不足、文件消失、读取失败、写入失败、同步失败、正式改名失败和源删除失败。错误信息面向用户使用中文摘要，同时保留技术原因供测试日志查看。

空间检查只是预警，不能替代写入错误处理，因为可用空间可能在任务中变化。

## 8. 分段交付

### M1A：选择与轻量写操作

- 长按多选、全选和取消
- 新建文件夹
- 单项目重命名
- 名称校验与同名提示
- 操作后刷新且保持合理目录位置

### M1B：复制与移动

- 目标文件夹选择模式
- 文件和多层文件夹递归复制
- 同存储快速移动与复制后删除移动
- `.part`、进度、取消和清理
- 同名冲突四种决定及应用到全部
- 空间不足、权限和非法目标提示

### M1C：删除与完整可靠性

- 多选递归永久删除
- 项目数和大小预扫描
- 二次确认与根目录保护
- 部分成功和失败详情
- 连续任务、取消恢复、内存和大目录测试

每个增量只构建、安装 Debug；M1 全部验收后再统一更新版本号和 `releases` 阶段包。

## 9. 测试与验收

所有破坏性测试仅在 `/storage/emulated/0/Download/WatchFilesTest/M1Sandbox` 中进行，不接触重要照片、录音或系统目录。

必须覆盖：

- 空文件、普通文件和大文件
- 空目录、三层以上嵌套目录和混合文件目录
- 多选文件与文件夹
- 同名文件、同名目录、文件与目录类型冲突
- 保留两者、替换、跳过和取消
- 复制中取消、移动复制阶段取消、删除中取消
- 目标位于源目录内部
- 空间不足和不可写目标
- 操作过程中源文件消失
- 源删除失败导致的部分成功
- 连续执行多次任务后没有持续内存上涨
- 操作结束后目录内容、选中状态和列表刷新正确

M1 总体验收条件：

- 中断或失败复制不破坏源文件。
- 移动只在目标成功后删除源项目。
- 不发生静默覆盖或静默永久删除。
- `.part` 清理仅作用于本任务可确认拥有的临时文件。
- 单任务状态和所有失败结果在圆屏上可理解、可返回。

## 10. 已确认决策

- 文件夹操作支持递归子目录。
- 同名冲突提供保留两者、替换、跳过和取消；替换需要再次确认。
- 删除为永久删除并二次确认，不实现回收站。
- 多选通过长按进入，单击切换选择。
- 复制与移动使用独立目标文件夹选择模式。
- M1 只保证前台进程存活期间执行；后台延续留给 M3。
- 采用独立文件操作引擎和协调器，不把写操作塞入现有浏览 ViewModel。
